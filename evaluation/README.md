# Evaluation

This directory contains scripts and results for evaluating RoSHI against OptiTrack motion capture ground truth.

## Metrics

- **MPJPE** (Mean Per-Joint Position Error, cm): Euclidean distance between predicted and GT joint positions in world coordinates.
- **JAE** (Joint Angle Error, degrees): Geodesic angle between predicted and GT parent-relative joint rotations (root-invariant).
- **MPJAR** (Mean Per-Joint Angular Error, degrees): Angle between predicted and GT bone direction vectors.
- **Recall**: Percentage of GT frames with a valid prediction (relevant for SAM3D which may miss detections).

## Methods Compared

| Method | Description |
|--------|-------------|
| `imu_naive` | IMU-only pose with naive global positioning |
| `imu_egoallo` | IMU joint rotations + EgoAllo-estimated root position |
| `egoallo` | EgoAllo egocentric pose estimation (no IMU) |
| `imu_tto` | RoSHI: IMU + test-time optimization (ours) |
| `sam3d` | SAM-3D-Body single-image reconstruction |

## Reproduction Steps

### Step 1: Fit SMPL-X to OptiTrack Ground Truth

```bash
python evaluation/fit_smplx.py --datasets dataset1 dataset2 dataset3 dataset4 dataset5 dataset6 dataset7 dataset8
```

This reads OptiTrack ROS bags and optimizes SMPL-X body parameters using 3-stage Adam optimization (global alignment в†’ body pose в†’ refinement). Outputs are saved to `evaluation/optitrack_gt_data/smplx_fits/`.

### Step 2: Compute Overall Metrics

```bash
python evaluation/compute_metrics.py
```

Options:
- `--datasets dataset1 dataset2 ...`: Select specific datasets
- `--methods imu_naive imu_tto ...`: Select specific methods
- `-o evaluation/metrics.md`: Export results to markdown

### Step 3: Compute Per-Activity Metrics

```bash
python evaluation/compute_metrics_by_activity.py
```

This uses `sequence_splits.json` to split datasets by activity transition points and computes per-activity metrics. Results are saved to `evaluation/metrics_by_activity.md`.

Options:
- `--export-dir evaluation/by_activity`: Save per-activity NPZ files
- `--no-export`: Only compute metrics, don't export NPZ

## Scripts Reference

| Script | Purpose |
|--------|---------|
| `compute_metrics.py` | Overall MPJPE/JAE/Recall per method and dataset |
| `compute_metrics_by_activity.py` | Per-activity metrics using sequence_splits.json |
| `fit_smplx.py` | Fit SMPL-X to OptiTrack skeleton data |
| `eval_utils.py` | Shared utilities (loading, coordinate transforms, synchronization) |
| `visualize_*.py` | Visualization scripts for each method's outputs |
| `generate_*.py` | Video generation for qualitative comparison |
